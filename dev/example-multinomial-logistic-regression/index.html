Multinomial logistic regression

In this example, we fit a Bayesian multinomial logistic regression model with the canonical link function.

Suppose that we are given a matrix of features X and a column vector of labels y. X has n rows and p columns. y has n elements. We assume that our observation vector y is a realization of a random variable Y. We define Œº (mu) as the expected value of Y, i.e. Œº := E[Y]. Our model comprises three components:

  1. The probability distribution of Y. We assume that each Y·µ¢ follows a multinomial distribution with k categories, mean Œº·µ¢, and one trial.
  2. The systematic component, which consists of linear predictor Œ∑ (eta), which we define as Œ∑ := XŒ≤, where Œ≤ is the column vector of p coefficients.
  3. The link function g, which provides the following relationship: g(E[Y]) = g(Œº) = Œ∑ = XŒ≤. It follows that Œº = g‚Åª¬π(Œ∑), where g‚Åª¬π denotes the inverse of g. In multinomial logistic regression, the canonical link function is the generalized logit function. The inverse of the generalized logit function is the softmax function. Therefore, when using the canonical link function, Œº = g‚Åª¬π(Œ∑) = softmax(Œ∑).

A multinomial distribution with one trial is equivalent to the categorical distribution. Therefore, the following two statements are equivalent:

  • Y·µ¢ follows a multinomial distribution with k categories, mean Œº·µ¢, and one trial.
  • Y·µ¢ follows a categorical distribution with k categories and mean Œº·µ¢.

Observe that the logistic regression model is a special case of the multinomial logistic regression model where k = 2.

In this model, the parameters that we want to estimate are the coefficients Œ≤. We need to select prior distributions for these parameters. For each Œ≤·µ¢ we choose a normal distribution with zero mean and unit variance. Here, Œ≤·µ¢ denotes the ith component of Œ≤.

We define this model using the Soss probabilistic programming library:

m = @model X,pool begin
    n = size(X, 1) # number of observations
    p = size(X, 2) # number of features
    k = length(pool.levels) # number of classes
    Œ≤ ~ Normal(0.0, 1.0) |> iid(p, k) # coefficients
    Œ∑ = X * Œ≤ # linear predictor
    Œº = NNlib.softmax(Œ∑; dims=2) # Œº = g‚Åª¬π(Œ∑) = softmax(Œ∑)
    y_dists = UnivariateFinite(pool.levels, Œº; pool=pool) # `UnivariateFinite` is mathematically equivalent to `Categorical`
    y ~ For(j -> y_dists[j], n) # `Y·µ¢ ~ UnivariateFinite(mean=Œº·µ¢, categories=k)`, which is mathematically equivalent to `Y·µ¢ ~ Categorical(mean=Œº·µ¢, categories=k)`
end;

iris = dataset("datasets", "iris");

label_column = :Species

Convert the Soss model into a SossMLJModel:

model = SossMLJModel(;
    model       = m,
    predictor   = MLJBase.UnivariateFinite,
    hyperparams = (pool=iris.Species.pool,),
    infer       = dynamicHMC,
    response    = :y,
);

Create an MLJ machine for fitting our model:

mach = MLJBase.machine(model, iris[!, feature_columns], iris[!, :Species])
